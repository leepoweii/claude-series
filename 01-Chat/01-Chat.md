# Claude Chat 學員講義

> 這份講義整理了 Claude Chat 的基礎操作、Connector 連接方式，以及 Artifact 的實作案例。

---

## 這份講義會帶你做到什麼

- 認識 Claude Chat 的主要介面
- 知道 `Project`、`Pin`、`Artifact`、`Incognito chats`、`Skills` 的用途
- 連上 Google Calendar Connector，讓 Claude 幫你讀取與新增行事曆
- 理解 Connector 權限設定的差異
- 上傳資料給 Claude，請它協助整理與分析
- 產出一個可互動、可分享的 Artifact

這份教材適合：

- 第一次接觸 Claude 的學員

## 課前準備

開始前，請先確認你有以下條件：

- 一個 Claude 帳號
- 一個 Google 帳號
- 可以登入 Claude 網頁版或桌面版
- 已下載這份教材附帶的資料檔，或能自行準備 CSV 檔案

這份案例會用到兩份資料：

- `餐飲業登記.csv`
- `家庭收支.csv`

這兩份資料都來自台灣政府公開資料。

## 第一部分：Claude Chat 與 Connector

這一部分的目標，是先讓你把 Claude 從聊天工具用成工作工具。

當你第一次打開 Claude 時，先認識幾個最常用的地方：

- 左側 `sidebar`：查看聊天紀錄，切換不同對話
- `Pin`：把重要對話固定在上方
- `Projects`：把相關對話整理在一起
- `Artifact`：查看 Claude 幫你產出的互動成果
- `Customize`：設定偏好、管理 Connectors
- 右上角小鬼圖案：`Incognito chats`
- 輸入框的 `+`：上傳檔案、選 Skills、設定工具

![[chat_ui.png]]

有幾個小地方先記住就好：

- `Incognito chats` 不會保留聊天紀錄，也不會使用你的 Memory
- 模型建議選 Sonnet，不會消耗太快，也足夠聰明
- 語音功能目前不是這份教材的重點，可以先略過

接著，我們來做第一個最重要的實作：把 Claude 連上 Google Calendar。

操作步驟如下：

1. 打開 Claude
2. 進入 `Customize`
3. 找到 `Connectors`
4. 點選 `Browse Connectors`
5. 找到 `Google Calendar`
6. 按下新增並完成 Google 授權
7. 回到 Claude，確認 Connector 已成功連線

連線完成後，你會看到幾種權限設定：

- `Always allow`：Claude 執行這類操作時不再詢問
- `Needs approval`：每次操作前都先詢問你
- `Blocked`：禁止 Claude 使用這項能力

剛開始使用時，建議你對寫入、修改、刪除這類操作保守一點，等到比較熟了才可以針對一些確定不會出錯的開 Always allow。

完成設定後，可以直接做兩個練習。

先把下面這段話貼進 Claude，請它幫你讀取下週行程：

```text
下週會有什麼行程安排？
如果我想安排一天下午去咖啡廳讀書，哪天比較有空？
```

這一步的重點，是讓你感受到 Claude 不是只在猜你想聽什麼，而是真的透過 Connector 去讀 Google Calendar。

接著，再請 Claude 幫你新增一筆行事曆：

```text
5/1 下午 3 點搭飛機回金門，幫我加入行事曆
預計 4 點到金門
```

完成後，再請 Claude 查一次 5/1 的行程，或直接打開 Google Calendar 檢查，確認它真的有寫進去。

做到這裡，你已經完成了這部分最重要的目標：

- 看懂 Claude Chat 的基本介面
- 成功連上第一個 Connector
- 體驗 Claude 讀取與寫入工具資料的方式

## 第二部分：Artifact 與資料分析

當文字沒辦法滿足你，或是你需要把成果整理給別人看時，通常就會進入 Artifact 的使用情境。

可以把 Artifact 理解成一個迷你的互動式網頁。它不只是展示結果，也可以加入拉桿、篩選器或其他互動元件，讓你繼續調整分析方式。

這一部分我們用一個公開資料案例來練習。問題是：

> 如果我想開一間特色咖啡廳，哪個縣市競爭比較少，而且消費力也夠？

這個案例會用到兩份資料：

- `餐飲業登記.csv`：台灣其他餐飲業登記清單，包含飲料店、咖啡廳等資料
- `家庭收支.csv`：各縣市平均每戶所得資料

你要練習的，不只是把資料丟給 Claude，而是學會一個比較穩定的資料處理流程：先分別處理資料，再合併分析，最後產出互動式 Artifact。

先開一個新對話，輸入：

```text
我考慮要開一間特色咖啡廳，哪邊的競爭最少，消費力又夠？
我會上傳資料請你協助分析。
```

接著上傳 `餐飲業登記.csv`，再輸入：

```text
這是全台灣其他餐飲業（含飲料店、咖啡廳）的登記清單。
公司地址欄位裡有縣市，幫我：
1. 從地址萃取縣市
2. 統計各縣市的登記數量
3. 先別做圖，把表格給我看
```

這一步很重要，因為它代表你不是一開始就叫 AI 全部做完，而是先讓它交出中間結果。這樣比較容易掌握方向，也比較容易修正。

接著再上傳 `家庭收支.csv`，然後輸入：

```text
這是各縣市平均每戶所得。
把這份資料和剛才的餐飲業數量合併，做成一個四象限的圖表
讓我找出競爭數量少、平均所得高的縣市，

把成果做一個 Artifact 圖表，同時顯示兩個維度。
```

Claude 會產出一個圖表，這裡最重要的不是哪個縣市最後看起來最好，而是你已經完成了一整個分析流程：

- 資料整理
- 跨資料合併
- 互動式呈現

接下來，你可以繼續要求 Claude 修改 Artifact。例如：

```text
幫我加一個拉桿 / 篩選器，讓我可以自己篩選所得範圍。
```

你也可以再追加條件，例如只顯示某個所得以上的縣市，或再加一個餐飲業數量的篩選器。這一步想建立的觀念是，Artifact 不是一次生成就結束，而是可以持續修改，最後變成一個真的能拿去討論的工具。

但這邊要提醒一下，若是想要分享連結給別人，建議你先開無痕頁面實際測一次，確認沒登入你帳號是否真的能打開。

做到這裡，你已經完成了這部分最重要的目標：

- 把資料交給 Claude 整理與分析
- 產出互動式 Artifact
- 反覆調整成果
- 把成果分享出去

## 本章重點整理

- Claude Chat 的價值不只是聊天
- Connector 讓 Claude 可以在你授權後讀取與操作工具
- Calendar 是理解 Connector 最簡單的第一步
- Artifact 讓 Claude 的產出從文字變成可互動成果
- 做資料分析時，先整理、再合併、最後視覺化，通常會比一次叫 AI 全做完更穩定
