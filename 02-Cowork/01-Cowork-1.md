## Cowork-1

# Cowork-1：Context 管理示範

> 這一章會帶你進入 Claude 的 Cowork 模式。
> 如果說 Chat 比較像和 AI 對話，Cowork 就更像讓 AI 真的在你的電腦上一起工作。

---

## 這一章會帶你做到什麼

- 理解 Chat 和 Cowork 的差別
- 看懂 Cowork 介面中幾個重要區塊
- 建立一個本機 Project，讓 Claude 直接讀寫你的資料夾
- 理解 Cowork 的權限設定
- 用一組真實的 Context 檔案，示範 AI 有沒有脈絡時，產出會差多少

---

## 適合誰

這一章適合：

- 已經用過 Claude Chat，想進一步把 AI 用成工作夥伴的人
- 想讓 AI 不只是聊天，而是能接手整理資料、提出企劃、參考本機檔案的人
- 想學會怎麼準備 Context，讓 AI 產出更貼近自己工作情境的人

---

## 課前準備

開始前，請先確認你有以下條件：

- 已完成 Chat 章節
- 有 Claude Pro 帳號
- 已安裝 Claude Desktop
- 已下載這章附帶的 Context 練習資料

這次會用到的示範資料，是一組「半畝咖啡」的假資料，包含：

- 品牌故事
- 菜單
- 月營運數據
- 活動紀錄
- 供應商資訊
- 交貨紀錄與對帳單

## 開場先埋一個 Schedule

在正式開始這一章之前，我很建議你先做一件事：先埋一個簡單的 `Schedule`。

原因很單純，因為排程這種東西不是設定完當下最有感，而是過幾個小時、過一天之後回來看成果，才會真正理解它的價值。

所以這裡我們先不要展開細節，也先不要花太多時間講排程設計。先把它當成一個伏筆，等學到後面的 `Dispatch + Schedule` 章節時，再回來驗收它到底跑出了什麼。

如果你想先埋一個示範用的排程，可以先設定這樣的任務：

```text
請每小時查詢一次最新的咖啡產業趨勢、咖啡消費變化、新品風向、門市經營做法、咖啡品牌行銷案例或相關研究資料。

先判斷本次是否有值得記錄的新趨勢主題。如果沒有明顯新資訊，請簡短記錄「本輪無明顯新趨勢」，不要硬寫內容。

如果有值得使用的趨勢，請先用 3-5 句話整理：
1. 趨勢是什麼
2. 為什麼值得注意
3. 和咖啡店經營、菜單、內容行銷或顧客體驗有什麼關聯

接著，請根據同一個趨勢主題，產出以下 3 個版本：

第一個版本：Instagram 行銷促銷貼文
- 目標是幫我們促銷產品或活動
- 要把趨勢轉成適合顧客理解的切角
- 要搭配一個基本促銷方案，例如限時折扣、組合優惠、加價購、第二杯優惠、期間限定品項等
- 內容請包含：貼文標題、貼文內文、促銷機制、CTA

第二個版本：產業知識分享貼文
- 目標是建立品牌的專業與權威感
- 請用知識型內容角度解釋這個趨勢
- 需要引用資料來源或文獻觀點，並清楚標註依據
- 內容請包含：主題標題、重點摘要、貼文內文、引用來源

第三個版本：內部知識分享版本
- 目標不是發文，而是讓內部同事知道這個趨勢可以怎麼用
- 無論這次有沒有適合直接發文，都一定要產出這個內部版本
- 請把趨勢轉成日常可執行的方法，例如產品調整、話術建議、陳列方式、活動想法、點餐推薦、社群觀察方向
- 內容請包含：趨勢摘要、對門市或品牌的意義、3 個可執行建議

整體語氣請務實、具體，不要寫成空泛模板。如果資料不足，請誠實說明不確定之處。
```

現在先把它設下去就好。後面課程走到 `Dispatch + Schedule` 的時候，再回來看它跑出了哪些結果、值不值得保留、要不要再調整。

## 第一部分：Cowork 與本機 Context

在 Chat 裡，Project 比較像雲端中的對話整理空間；在 Cowork 裡，Project 比較像一個真的連到你電腦資料夾的工作區，你可以理解成你跟 AI 助手共用的一個辦公桌或資料夾。

這是兩者最本質的差別：

- Chat 的 Project 活在雲端
- Cowork 的 Project 手腳長在你的電腦裡

也因為這樣，Cowork 才會更像一起工作的夥伴。它不是只根據你丟進對話框的內容回答，而是可以直接讀你資料夾裡的檔案，依照那些脈絡來工作。

打開 Cowork 後，你會看到幾個和 Chat 不太一樣的地方：

- `New Task`：在這裡開一個新的工作，而不是新的聊天
- `Projects`：管理本機工作專案
- `Schedule`：安排 Claude 定期執行任務
- `Live Artifacts`：持續更新的互動成果
- `Dispatch`：之後章節會再談

這一章最重要的不是把每個按鈕都記住，而是先抓住一個核心觀念：

> Cowork 的價值，不是功能比較多，而是它真的能讀寫你電腦上的資料。

接下來，我們先建立一個本機 Project。

你可以先在桌面建立一個資料夾，例如：

```text
半畝咖啡
```

這個動作可以把它想成：你找了一位新同事一起工作，第一件事不是叫他開始做事，而是先把工作資料整理到同一個地方。

接著在 Claude Desktop 的 Cowork 裡：

1. 進入 `Projects`
2. 點 `New Project`
3. 選擇 `add a local folder`
4. 選擇你剛才建立的 `咖啡廳工作` 資料夾

建立 Project 時，你可能會看到幾個設定欄位，例如 `Instructions`、`Context`、`Pin` 等。

這一章建議先用最單純的方式開始：

- `Instructions` 可以先留白
- `Context` 不用額外加太多來源
- 先把重點放在資料夾本身的內容

這樣做是因為本章要示範一個很重要的觀念：網路上很多人在分享神奇的 Prompt 大全，並不是真的有什麼神奇能量，實際上，他就是把 Context 寫在 Prompt 裡面而已，很多時候，真正決定產出品質的不是你寫了多華麗的指令，而是你有沒有把對的 Context 準備好。

在 Cowork 裡，這比較像是整個 Project 的工作邊界：

- `Ask`：Claude 每次操作前先詢問你
- `Always allow`：Claude 可以直接操作

建議先保留 `Ask`。因為 Cowork 能做的事比單純聊天多很多，剛開始先讓每一步可見，會比較安全。

## 第二部分：Context 管理示範

這一部分是這章真正的重點。

很多人學 AI 的時候，會把焦點放在「怎麼寫 prompt」。但在這個示範裡，你會看到一件更重要的事：

> Prompt 的重點，不是寫得多厲害，而是你有沒有把工作脈絡交給 AI。

先把教材附的半畝咖啡檔案放進 `咖啡廳工作` 這個資料夾。這些檔案的作用，不是單純讓 AI 讀資料，而是讓它知道：

- 你是什麼品牌
- 你賣什麼
- 你過去做過哪些活動
- 你的店大概是什麼規模
- 你的營運數字大概在哪裡

把資料放進去之後，可以先做兩個小測試，確認 Claude 真的讀得到。

先試試看：

```text
幫我列出這個 Project 資料夾裡有哪些檔案
```

再試試看：

```text
我們的菜單有什麼特色？
```

如果 Claude 能回答出來，就代表它已經讀到這些本機 Context。

接下來我們一起做個實驗，請準備兩個畫面：

- 一邊是網頁版 Claude Chat 的無痕對話
- 一邊是剛才已經讀到 Context 的 Cowork

然後兩邊都輸入同一句話：

```text
幫我設計一個聖誕節促銷活動，請先跟我提問確認細節，產出一份企劃書給我
```

這裡有一個很有趣的觀察點：

- Chat 會像商學院學生做報告，CoWork 會明顯像是團隊的夥伴提出來的草稿。

沒有 Context 的 Chat，通常只能做出一份很像模板的企劃書。它可能句子很漂亮，架構也完整，但本質上不知道你是誰，也不知道你的店在哪裡、內容跟規模可能都跟你沒關係。

有 Context 的 Cowork 則不同。它會去讀品牌故事、菜單、活動紀錄、營運數據，然後把這些脈絡帶進提案裡。

所以你最後看到的差異，通常會長這樣：

- Chat 產出的內容比較空泛，像通用模板
- Cowork 產出的內容比較具體，會反映品牌背景與營運限制
- Chat 容易給出不切實際的大預算
- Cowork 比較有機會給出貼近真實規模的方案

這個示範最想讓大家理解的是：

> Context 設計好，不需要神奇的 Prompt 咒語，就可以讓 AI 真的做出能落地的成果。

如果你的資料脈絡準備得夠完整，有時候一句很普通的交辦就夠了。

做到這裡，你已經完成這一章最重要的學習目標：

- 知道 Cowork 的 Project 是本機工作區
- 知道 Context 不是裝飾，而是工作脈絡
- 看到有沒有 Context，會如何直接改變 AI 的產出

---

## 本章重點整理

- Cowork 和 Chat 最大的差別，在於 Cowork 能直接碰到你電腦裡的資料
- 建立 Project 的第一步，不是先想 prompt，而是先整理工作資料夾
- `Ask` 和 `Always allow` 是 Cowork 的工作權限設定，和 Connector 權限不同層
- Context 的重點不是節省澄清對話，而是讓產出更貼近真實情境
- 如果資料夾準備得夠完整，一句簡單的 prompt 也能得到很實用的結果

---

## 一句話總結

> Cowork 的關鍵，不是叫 AI 幫你做更多事，而是先把你的工作脈絡整理好，讓 AI 真正知道自己在幫誰工作。
