## 第一部分：Chrome 與重複工作的處理

這一章的起點，是一個很常見的工作情境：月底對帳。

很多人看到 AI 時，第一反應是拿來問問題、寫文案、做企劃。但在實際工作中，還有一大類事情同樣很值得交給 AI，那就是重複性的流程工作。

例如：

- 每個月都要檢查廠商寄來的對帳單
- 都要把對帳單和交貨紀錄比對一次
- 都要確認有沒有多報、漏報或單價差異
- 如果有問題，還要再整理一段訊息去聯絡業務

這些步驟單看都不難，但每次都重做一次很花時間。這就是 Cowork 很適合發揮的地方。

在正式做 Skill 之前，我們要先寄出測試用的對帳單，請到 /Attachments/emails 裡面，將對帳單A-島嶼咖啡供應-202609.pdf 用 Email 寄給自己。

接著讓 Claude 手動跑完整個流程一次。

先進到半畝咖啡的 Project，然後確認 Gmail Connector 已經連上。可以直接對 Claude 說：

```text
幫我從 Gmail 檢查有沒有廠商寄來的對帳單。
我要核對貨款，準備結帳。
```

如果信箱裡有符合條件的信件，Claude 會先從 Gmail 裡把它找出來。

但這裡會遇到一個限制：Gmail Connector 不沒辦法直接把 PDF 附件內容完整拿來讀。

這時候，就會進到這一章的另一個重點：`Claude for Chrome`，讓 Claude 透過 Chrome 來查看 PDF。

當 Claude 成功打開 Gmail 裡的 PDF 對帳單後，你就可以接著交辦任務：

```text
將它跟我們的交貨紀錄比對
```

這時候 Claude 會開始把 PDF 裡的帳單內容，和 Project 裡原本的交貨紀錄交叉比對。

這一步的重點不是讓 AI 完全取代你，而是讓它先把可疑之處找出來，再由你確認。因為 AI 本質還是文字接龍大師，AI 可能幫你找到問題，也可能自己看錯，所以最後還是要有人做一次判斷，只是可以節省很多時間。

如果 Claude 找到了差異，接下來你可以再說：

```text
我們平常是用 LINE 對話，幫我擬一份 LINE 訊息跟他核對。
```

這樣它就能順勢把發現的問題整理成一段可以直接拿去傳的訊息草稿。

做到這裡，其實你已經完成了一個很完整的流程：

- 從 Gmail 找到對帳單
- 透過 Chrome 讀取 PDF
- 對照本機交貨紀錄
- 找出差異
- 草擬聯絡訊息

這正是 Skill 最適合接手的類型。

## 第二部分：把流程做成 Skill

當你發現某件事每個月都會重複做一次，就很適合把它整理成 Skill。

在這裡，Skill 可以先把它想成：

- 一份工作 SOP
- 一份預存好的操作說明
- 一份讓 Claude 知道「接下來這類任務要怎麼跑」的 markdown 規則檔

也就是說，Skill 的價值不只是省下重打 prompt 的時間，而是把整個做事流程標準化。

做法可以很直接。當你已經手動跑完一次流程後，對 Claude 說：

```text
幫我釐清剛剛的工作流程，列成條列式的 SOP
```

這裡如果你有安裝官方的 `Skill Creator` plugin，就可以讓 Claude 協助把這個流程整理成一份 Skill。

整理完成後，你通常會看到一份可預覽的內容。這時候很值得停一下，看看它到底長什麼樣子。因為這一刻很關鍵：

> Skill 並不是什麼神秘黑盒子，它本質上就是一份 markdown 形式的工作說明。

這也代表兩件事：

- 你可以自己改
- 你也可以叫 Claude 幫你改

之後再遇到類似的工作，你就不需要從頭講整個流程，而是可以直接呼叫這個 Skill。

接下來，你可以用附件的第二張對帳單再測一次，看看 Skill 能不能把整套流程重新跑完。

如果它運作正常，你會發現一件小問題，Claude 會重複處理它已經看過的對帳單。

這時候，就可以再透過對話，請 Claude 把 Skill 更新成第二版，加入一個「對帳紀錄表」的概念：

- 開始前先讀取對帳紀錄
- 已經處理過的對帳單就跳過
- 結束後把本次結果寫回紀錄表

這樣一來，Skill 就不只是「會跑流程」，而是開始有一點真正工作系統的味道。它知道哪些事情做過，哪些還沒做，也更適合長期使用。

記住真正有用的 Skill，不是第一次就寫到最好，而是會在使用中慢慢迭代。

這一章最想讓你建立的觀念是：

> Skill 不是把 prompt 存起來而已，而是把工作方法整理成可以反覆執行、反覆修改的流程。

做到這裡，你已經完成這一章最重要的學習目標：

- 知道 Claude for Chrome 是 Connector 不夠用時的重要補位工具
- 知道怎麼讓 Claude 手動跑一次完整流程
- 知道怎麼把流程整理成 Skill
- 知道 Skill 可以根據實際使用情況持續迭代

---

## 本章重點整理

- Skills 最適合處理重複性工作
- Claude for Chrome 不是附帶細節，而是很多真實流程裡的重要補位能力
- 對帳這種任務很適合先讓 AI 找差異，再由人做最後確認
- Skill 的本質是 markdown 形式的 SOP，不是黑盒子
- 第一版 Skill 幾乎一定還要再調整，真正的價值在迭代

---

## 一句話總結

> 當你把一個重複流程先做一次、再整理成 Skill，Claude 就開始從「幫你一次」變成「以後都能照這個方法幫你做」。
