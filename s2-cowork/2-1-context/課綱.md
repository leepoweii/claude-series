---

## 核心概念：為什麼是 Local Folder？

> Cloud Project（上傳到 Knowledge）：AI 讀一次，靜態快照，不能存檔
>
> **Local Folder Project：AI 有持續的讀寫能力——可以讀你的資料、存報告、更新紀錄**
>
> 這才是「AI 在你電腦上工作」的感覺，後面每一集都建立在這個基礎上。

## CoWork 介面導覽（1.5 min）

帶過主要區域，不用停太久：

| 區域 | 說明 |
|---|---|
| Projects 列表 | 每個 Project 是一個獨立工作空間 |
| Project Folder | **本機資料夾路徑**，AI 可以直接讀寫 |
| Instructions | 給這個 Project 的固定指令（後面集數會用到）|
| Chat | 在這個 Project 內的對話 |

> 📌 重點：**「AI 能讀你電腦上的檔案」**——這是 Chat 做不到的

## 建立 Local Folder Project（2.5 min）

### Step 1 — 在桌面建立資料夾

```
桌面 → 新增資料夾 → 命名「半畝咖啡」
```

把 GitHub 下載的 Context 檔案放進去：
- `半畝咖啡-品牌故事.md`
- `半畝咖啡-菜單.csv`
- `半畝咖啡-月營運數據.csv`
- `半畝咖啡-活動紀錄.csv`
- `半畝咖啡-供應商.md`

> 📌 這個資料夾就是「AI 的辦公桌」——他能看到的、能存的，都在這裡

---

### Step 2 — CoWork 指定 Folder

1. Claude Desktop → CoWork → New Project
2. 命名「半畝咖啡」
3. **Project Folder → 選取桌面的「半畝咖啡」資料夾**
4. 確認 AI 可以存取

> 📌 讓觀眾看到：AI 現在有了一個本機地址，不只是上傳給他看而已

---

### Step 3 — 驗證 AI 能讀檔

```
幫我列出這個 Project 資料夾裡有哪些檔案
```

AI 讀取後回傳清單。讓觀眾確認 AI 真的看到了本機檔案。

## 有/無 Context 對比 Demo（3.5 min）

**先在一般 Chat（沒有 Project）問：**
```
幫我設計一個國慶連假的快閃活動
```
→ AI 給通用、模板式的答案

**再在 Project 裡問同樣的問題：**
```
幫我設計一個國慶連假的快閃活動
```
→ AI 提到：金門觀光客比例、高粱拿鐵招牌品項、過去活動成效、人力限制

> 📌 這個對比是整集最重要的一刻，要給觀眾時間感受差異

## 講師口語備忘

**開場：**
「Chat 是跟 AI 聊天，問一次答一次，關掉就忘了。CoWork 不一樣——你建一個 Project，指定你電腦上的一個資料夾，AI 就可以直接讀寫那個資料夾。他不只是記住你說了什麼，他可以真的幫你存檔、整理資料。」

**介紹 Local Folder 概念時：**
「為什麼要用本機資料夾，而不是直接上傳到雲端？因為上傳是靜態的——你給他看，他看完就沒了。本機資料夾是動態的——他可以存東西進去，下次開啟還在，累積起來。後面幾集你會看到，他幫你存的報告、對帳結果，都會出現在這個資料夾裡。」

**建立資料夾時：**
「這個資料夾就是 AI 的辦公桌。他能看到的、能寫的，都在這裡面。你把公司的資料放進去，他就認識你的公司了。」

**CoWork 指定 Folder 後：**
「好，現在讓我們確認他真的看到了——」（輸入「列出資料夾裡的檔案」）「你看，他回傳了清單，這些就是我們放進去的 Context 檔案。」

**對比 Demo 後：**
「你看到差異了嗎？沒有 Context 的時候，他給你的是任何一間咖啡廳都可以用的答案。有了 Context，他知道你的招牌品項、知道你的客群、知道你上次做什麼活動——這才是真正的 AI 協作。」

**GitHub 說明：**
「今天示範的是我事先準備好的半畝咖啡，但你的公司當然不一樣。GitHub 上有一個 prompt，下載回去貼給 AI，他會一步一步問你問題，幫你整理出自己公司的 Context，最後存成 .md 放進你的 Project 資料夾就好。」
