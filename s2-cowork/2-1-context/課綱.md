## CoWork 介面導覽（2 min）

從 Claude Desktop 打開 CoWork，帶過主要區域：

| 區域 | 說明 |
|---|---|
| Projects 列表 | 每個 Project 是一個獨立工作空間 |
| Project 內：Knowledge | 上傳的文件、Context 都在這裡 |
| Project 內：Instructions | 給這個 Project 的固定指令 |
| Project 內：Chat | 在這個 Project 內的對話 |
| 跟 Chat 的差異 | Chat 關掉就忘，Project 的 Knowledge 永遠在 |

> 📌 重點說明：**Knowledge 跨對話，Chat 不跨對話**

---

## 建立 Project（2.5 min）

1. 點左上角 New Project
2. 命名：「半畝咖啡」
3. 進入 Project
4. 點 Knowledge → Upload Files

**上傳的檔案（已在 repo）：**
- `半畝咖啡-品牌故事.md`
- `半畝咖啡-菜單.csv`
- `半畝咖啡-月營運數據.csv`
- `半畝咖啡-活動紀錄.csv`
- `半畝咖啡-供應商.md`

---

## 有/無 Context 對比 Demo（3.5 min）

**先在一般 Chat（沒有 Project）問：**
```
幫我設計一個國慶連假的快閃活動
```
→ AI 給通用、模板式的答案

**再在 Project 裡問同樣的問題：**
```
幫我設計一個國慶連假的快閃活動
```
→ AI 提到：金門觀光客比例、高粱拿鐵招牌品項、過去活動成效、人力限制

> 📌 這個對比是整集最重要的一刻，要給觀眾時間感受差異

## 講師口語備忘

**開場：**
「Chat 是跟 AI 聊天，問一次答一次。CoWork 不一樣——你建一個 Project，把公司的資料放進去，AI 就會記住。每次在這個 Project 裡問問題，他都知道你是誰、你的公司是什麼。」

**建立 Project 時：**
「Project 名稱就是你公司的名字，這樣之後清楚。Knowledge 這個地方，就是你要給 AI 的背景資料——品牌故事、菜單、數據、供應商，全部丟進來。」

**上傳完畢後：**
「好，他現在認識半畝咖啡了。我們來測試一下。」

**對比 Demo 後：**
「你看到差異了嗎？沒有 Context 的時候，他給你的是任何一間咖啡廳都可以用的答案。有了 Context，他知道你的招牌品項、知道你的客群、知道你上次做什麼活動——這才是真正的 AI 協作。」

**GitHub 說明：**
「今天示範的是我事先準備好的半畝咖啡，但你的公司當然不一樣。GitHub 上有一個 prompt，你下載回去，貼給 AI，他會一步一步問你問題，幫你整理出自己公司的 Context。」
