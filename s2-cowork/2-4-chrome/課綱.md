# 2-4 Chrome — 讀取瀏覽器頁面

> Series 2：Claude CoWork（第四章節）
> 時長：10 分鐘
> 前置：完成 2-3、已安裝 Chrome 瀏覽器

## 學完你能做到

- 安裝 Claude Chrome Extension
- 用 Claude 讀取任何瀏覽器頁面的內容
- 把讀取到的資料整理成 CSV 格式（模擬營業日報）

> 不需要 API 串接，不需要工程師，直接讀。」

## Demo 流程

### Step 1 — 安裝 Chrome Extension（2 min）

1. 開 Chrome → Chrome Web Store
2. 搜尋「Claude」→ 找到官方 Anthropic Claude Extension
3. 點「加到 Chrome」→ 允許
4. 登入 Claude 帳號

> 📌 講師提示：如果 Chrome Web Store 搜尋不到，直接提供安裝連結

---

### Step 2 — 打開 POS Demo 頁面（30 sec）

打開 `pos-demo.html`（本地檔案或放到 GitHub Pages）：
```
file:///Users/.../pos-demo.html
```

頁面顯示：今日銷售紀錄（品項、數量、金額、時段）

---

### Step 3 — Claude 讀取頁面（2.5 min）

1. 頁面右上角點 Claude Extension 圖示
2. 點「讀取此頁面」或直接輸入問題
3. 輸入：
```
讀取目前頁面的銷售資料，整理成以下格式：
時段、品項、數量、單價、小計
```

AI 讀取頁面 HTML 內容，產出結構化資料。

---

### Step 4 — 整理成 CSV（3.5 min）

```
把這些資料整理成 CSV 格式，
加上今天日期，以及每個時段的小計和總營收。
格式我可以直接存成 .csv 貼到 Google Sheets。
```

AI 產出 CSV 格式，可以直接複製貼到試算表或存成檔案。

---

### Step 5 — 延伸說明（1 min）

說明其他可以用的場景：
- 訂單平台（Foodpanda、Uber Eats 後台）
- 網路銀行對帳頁面
- 任何有資料的網頁報表
- 電商後台（蝦皮、Shopee）

> 「只要頁面上看得到，他就能讀。當然，有些平台有隱私限制，讀取前要注意自己的使用條款。」

## 講師口語備忘

**開場：**
「你的 POS 系統有辦法直接下載今天的銷售記錄嗎？很多系統沒有，或者要另外付費。但沒關係，只要頁面上看得到，Claude 就能幫你讀出來。」

**安裝時：**
「Chrome Extension 裝一次，以後任何網頁都可以叫 Claude 讀。不只是 POS，任何你常用的系統後台都可以。」

**讀取頁面後：**
「你看，他讀到了頁面上的所有資料——品項、數量、金額都在。這個不是截圖 OCR，他讀的是頁面的文字結構，所以資料很乾淨。」

**CSV 產出後：**
「這個格式可以直接複製貼到 Google Sheets，或者存成 .csv。以後每天下班，打開頁面，一個指令，今天的營業日報就好了。」

**延伸說明時：**
「這個方法可以用在很多地方：訂餐平台後台、網路銀行的對帳頁面、電商後台。只要頁面上有資料，他就能讀。當然，要注意自己平台的使用條款，有些平台不允許自動化讀取。」

「我們在 Series 2 做了四件事：讓 AI 認識你的公司、做了一份企劃和海報、自動對帳、讀取 POS 資料。這些全部在 Claude CoWork 裡，不需要任何程式背景。接下來 Series 3，我們要做一件更酷的事——人不在辦公室，AI 幫你工作。」
