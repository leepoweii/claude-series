# 4-1 — Scheduled：每日工作系統

> Series 4：Claude Scheduled
> 時長：10 分鐘
> 前置：完成 Series 2-3（Project 已建立、Context 已載入）

---

## 學完你能做到

- 設定早上的自動排程：掃 Gmail + 行事曆，生成 Today's Focus .docx 存到 Project 資料夾
- 設定晚上的 Closedown 排程：讀取當天 .docx，整合進 Context + 累積 Work Log
- 建立一套「每天只要編輯那份 docx，記憶自動累積」的工作習慣

---

## 核心概念：雙向每日系統

```
早上 07:30（自動）
  → Gmail 掃昨晚到今早的信
  → 行事曆看今天行程
  → 生成 Today's Focus.docx → 存到 Project 資料夾

白天（你）
  → 打開 docx，編輯：補充昨天沒完成的、今天的決策、重要對話

晚上 22:00（自動）
  → 讀取今天的 docx
  → 把決策、重要事項 → 更新 Project Context
  → 把完成事項 → 累積到 Work Log
  → docx 歸檔（移到 /archive/YYYY-MM-DD/）
```

> 每天只要花 5 分鐘編輯 docx，AI 幫你把記憶累積起來。

## Demo 流程

### Step 1 — 說明系統（1.5 min）

用一張圖說明：早上 → 白天 → 晚上的循環。
重點：docx 是中間的介面，你只要管 docx，其他都是 AI 的事。

---

### Step 2 — 設定早上排程（3.5 min）

在 CoWork 輸入：
```
幫我設定一個每天早上 7:30 執行的排程任務，叫做「每日晨報」。

任務內容：
1. 透過 Gmail Connector 讀取過去 12 小時的新信件，摘要重要的
2. 透過 Google Calendar 看今天的行程
3. 結合 Context 裡的待辦事項，生成一份 Today's Focus 文件
4. 格式是 .docx，存到 Project 資料夾，檔名：Today's Focus - [日期].docx

文件格式：
# Today's Focus - [日期]

## 今日行程
[來自 Calendar]

## 重要來信
[Gmail 摘要]

## 今日重點任務
[結合 Context 的建議]

## 備忘 / 筆記
（這裡留白，等你手動填）
```

CoWork 設定完排程，確認時間和任務內容。

---

### Step 3 — 設定晚上 Closedown 排程（2.5 min）

```
再幫我設定一個每天晚上 22:00 執行的排程任務，叫做「每日 Closedown」。

任務內容：
1. 找到今天的 Today's Focus docx
2. 讀取我在「備忘 / 筆記」裡填寫的內容
3. 把重要決策和更新 → 更新到 Project Context 對應的部分
4. 把今天完成的事項 → 追加到 Work Log（格式：日期 + 摘要）
5. 把 docx 移到 archive 資料夾（建立 /archive/YYYY-MM-DD/）
```

---

### Step 4 — 示範一天後的結果（1.5 min）

模擬「跑了一天後」的狀態：
- 打開 Context，給觀眾看：有新的決策被記進去了
- 打開 Work Log，給觀眾看：今天完成的事有紀錄
- 打開 archive 資料夾：docx 已歸檔

> 📌 可以預先準備一個「已跑過一天」的 Project 狀態作為示範
